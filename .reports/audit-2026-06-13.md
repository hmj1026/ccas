### R01 · High · frontend-ui-ux · effort=small
**全應用缺少 React Error Boundary,lazy chunk 載入失敗或 render 例外導致整頁白屏**

- 位置:`/Users/paul/Project/ccas/frontend/src/App.tsx:61-119`
- 修法:建議採用最小侵入方案：

1. 安裝套件（可選，若偏好 class component 自行實作亦可）：
   `pnpm add react-error-boundary`

2. 在 App.tsx 頂層（`<QueryClientProvider>` 內、`<BrowserRouter>` 外或內均可）加入一個 ErrorBoundary，fallback 顯示「頁面載入失敗，請重新整理」+ 重試鈕；對 chunk load error（message 含 "Loading chunk" / "Failed to fetch dynamically imported module"）自動呼叫 `window.location.reload()`。

3. 若不引入套件，自行撰寫 class component：
   ```tsx
   class AppErrorBoundary extends React.Component {
     state = { error: null }
     static getDerivedStateFromError(error) { return { error } }
     componentDidCatch(error) {
       if (/Loading chunk|dynamically imported/i.test(error.message)) {
         window.location.reload()
       }
     }
     render() {
       if (this.state.error) return <ErrorFallback />
       return this.props.children
     }
   }
   ```

4. 包住 `<Routes>` 或整個 `<BrowserRouter>`，使所有路由均受保護。

5. 可選：React Router 7 data router（createBrowserRouter）模式下可改用 errorElement 每路由設置，但遷移成本較高，不建議為此單一問題重構路由層。

```python
import os

def write_private_token_file(path: Path, content: str) -> None:
    """以 0600 權限原子寫入 token 檔，避免 write→chmod race window。"""
    path.parent.mkdir(parents=True, exist_ok=True)
    flags = os.O_WRONLY | os.O_CREAT | os.O_TRUNC
    fd = os.open(path, flags, 0o600)
    try:
        with os.fdopen(fd, "w", encoding="utf-8") as f:
            f.write(content)
    except Exception:
        # fd 已被 fdopen 接管，fdopen 失敗時 fd 仍開著須關閉
        try:
            os.close(fd)
        except OSError:
            pass
        raise
    # 對抗部分系統 umask 仍影響 os.open 的防禦性 chmod
    os.chmod(path, 0o600)
```
